# 微信文章链接转换器 — 开发计划

## 目标

在现有 wechat-article-collector 项目上新增**批量链接转换**功能：用户粘贴多条微信文章链接，系统抓取并缓存为纯净 HTML，返回可被 NotebookLM/ChatGPT/Claude 等 AI Agent 直接浏览的公网 URL。部署到火山引擎 ECS（2C4G），通过 `http://<公网IP>:8686` 直接访问，无需 PDF/Markdown，轻量高效。

## 核心流程

```
用户粘贴 N 条微信链接
      ↓
POST /api/convert  (批量)
      ↓
逐条抓取 → 解析 → 下载图片 → 生成纯净HTML → 持久化到磁盘
      ↓
返回 N 条公网可访问 URL
      ↓
AI Agent 通过 GET /article/{id} 直接读取文章内容
```

---

## Phase 1: 后端 — 链接转换核心

### Step 1: 新建 `core/converter.py` — 纯净 HTML 生成器
- 复用 `core/parser.py` 的 `parse_article()` 解析结果
- 生成语义化 HTML5：UTF-8、无 JS、最小内联 CSS
- 图片路径替换为代理路径 `/article/{id}/images/{name}`
- 添加 `<meta>` 标签（title, description, author）便于 AI Agent 理解
- 文章 ID = 原始 URL 的 MD5 前 8 位

### Step 2: 新建 `core/cache.py` — 磁盘缓存管理

存储结构：
```
data/articles/{article_id}/
  ├── article.html   # 纯净 HTML 页面
  ├── meta.json      # 元数据（title, source, author, date, original_url, created_at）
  └── images/        # 下载的图片
```

接口：`save / load / list / is_cached / delete`，启动时扫描磁盘构建内存索引。

### Step 3: 修改 `app.py` — 新增 API 路由

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/convert` | 批量转换，接收 `{"urls": [...]}` |
| GET | `/article/{id}` | 返回纯净 HTML 页面 |
| GET | `/article/{id}/images/{filename}` | 返回缓存图片 |
| GET | `/api/cached` | 列出所有已缓存文章 |
| DELETE | `/api/cached/{id}` | 删除单条缓存 |

批量转换逐条处理，间隔 1-3s 防微信反爬。已缓存的 URL 直接跳过。

## 验证步骤

1. curl 发送真实微信链接到 `POST /api/convert`，验证返回格式正确
2. 浏览器打开 `http://localhost:8686/article/{id}`，页面渲染正常、图片可加载
3. `curl -s` 获取 HTML，确认无 JS 干扰，AI 可直接解析
4. 重启服务后已缓存文章仍可访问（持久化验证）
5. `docker compose up` 成功运行，端口可访问
6. 部署到 ECS 后从外网访问转换后的 URL

## 关键决策

- **保留原有功能**：Markdown/PDF/Obsidian 功能全部保留，链接转换作为独立新模块
- **MD5[:8] 作为文章 ID**：同 URL 不重复抓取
- **图片下载到本地代理**：避免微信 CDN 过期失效
- **批量顺序处理**：非并发，每条间隔 1-3s 防反爬
- **HTTP 协议**：无域名无法配 HTTPS，若 AI Agent 要求 HTTPS 后续加 Caddy + 域名

---

## 注意事项

1. **HTTPS 问题**：NotebookLM 可能要求 HTTPS URL 才能抓取。遇到时方案：购买域名 + Caddy 反向代理自动配 Let's Encrypt 证书
2. **磁盘清理**：初版不设自动清理，后续可加 30 天 LRU 策略（2C4G 磁盘有限）
3. **并发安全**：初版单用户场景不处理文件锁，后续按需加
